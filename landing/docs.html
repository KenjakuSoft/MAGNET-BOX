MagnetBox documentation

MagnetBox is a self-hosted app that turns magnets, torrents and direct links into clean HTTP downloads you can grab or stream in your browser — running entirely on a server you control.

It's free and open. You run it on your own hardware; nothing phones home, and no data leaves your machine. This guide explains why it exists, what it does, and how to get it running — even if you've never set up a server before.

💚 MagnetBox is donation-supported. If it's useful to you, consider supporting development.

Why MagnetBox

Most "torrent-to-direct-download" and streaming services are run by someone else: your links, your activity, and your data pass through their servers, usually behind a paid subscription. MagnetBox flips that around.

  • Your server, your data. Everything runs on hardware you own or rent. There's no third party in the middle.
  • No subscription, no lock-in. It's free software. Run it forever; nothing to renew.
  • One small binary. No database server, no container stack required — a single Rust executable plus a folder.
  • Private by design. Multi-user with invite-only registration, 2FA, and an admin console — built for a small, trusted group, not the open internet.

It's a neutral, general-purpose tool — like qBittorrent or Plex. Use it for content you have the right to download: your own files, Linux ISOs, Creative-Commons and licensed media, and anything else you're allowed to.

What it does

  • Add anything — paste a magnet, a .torrent URL, or any http(s) link; the type is auto-detected.
  • Download to your server at full speed, then grab a private direct link or play it in-app.
  • Stream while downloading — the built-in player seeks before the file finishes and loads subtitle tracks from the torrent.
  • Pick files inside a torrent, pause/resume/delete, batch-add, and drag-and-drop .torrent files.
  • Multi-user with invite-only registration, roles, and per-user limits.
  • Admin console — live CPU/RAM/disk, users, sessions, audit log, bandwidth, bulk controls.
  • Retention — optionally auto-delete content after N days to save space.
  • API — OpenAPI 3.1 spec + per-user tokens for scripts and media tools.

Quick start

If you just want to try it on your own computer:

# 1. install Rust (rustup.rs), then clone & run:
git clone https://github.com/KenjakuSoft/MAGNET-BOX.git
cd MAGNET-BOX
cargo run

It prints a URL (default http://127.0.0.1:8080) and, on first run, a generated admin password. Open the URL, sign in as admin with that password, and you're in.

⚠️ Run it in a terminal so you can see the first-run password. Lost it? See FAQ.

Build from source

MagnetBox is a Rust project. Install the Rust toolchain from rustup.rs, clone the repo, then build:

git clone https://github.com/KenjakuSoft/MAGNET-BOX.git
cd MAGNET-BOX

# development run
cargo run

# optimized production binary → target/release/magnetbox
cargo build --release

The web UI and docs are compiled into the binary, so the single executable is all you need to ship.

Deploy to a server

For a real, always-on instance you'll want a small Linux server (VPS) and a domain. Beginner-friendly path:

  1. Get a domain (Cloudflare, Porkbun, Namecheap) and a VPS running Ubuntu 24.04 (Hetzner, DigitalOcean, Vultr). 2 vCPU / 4 GB RAM is plenty; pick a host with generous bandwidth and enough disk for your downloads.
  2. Point the domain at the VPS with an A record → the server's IP.
  3. Build it on the server: 
    apt update && apt install -y build-essential pkg-config curl
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"
cargo build --release
  4. Run it as a service with the included deploy/magnetbox.service (systemd) — it binds localhost, enables secure cookies, and locks down the filesystem. Then systemctl enable --now magnetbox. Grab the first-run admin password with journalctl -u magnetbox.
  5. Add HTTPS with Caddy using the included deploy/Caddyfile — it gets a free certificate automatically and proxies to the app.
  6. Firewall: ufw allow OpenSSH && ufw allow 80 && ufw allow 443 && ufw enable.
  7. Open your domain, sign in, change the admin password, and turn on 2FA.
Keep the app bound to 127.0.0.1 behind Caddy — never expose the app port directly. The deploy/ folder has ready-to-use Caddyfile and systemd unit.

Configuration

MagnetBox is configured entirely with environment variables — no config file to edit.

VariableDefaultPurpose
MAGNETBOX_PORT8080Listen port
MAGNETBOX_BIND127.0.0.1Bind address (keep localhost; proxy in front)
MAGNETBOX_DIR./downloadsWhere downloaded files are stored
MAGNETBOX_DATA./magnetbox-dataWhere accounts/settings live
MAGNETBOX_HTTPS01 marks session cookies Secure (set when behind HTTPS)
MAGNETBOX_ADMIN_USERadminFirst-run admin username
MAGNETBOX_ADMIN_PASSWORDgeneratedFirst-run admin password (≥8 chars). Set it to reset a lost admin password.
RUST_LOGinfoLog verbosity

Adding content

Paste into the add bar at the top:

  • A magnet link, or an http(s) URL to a .torrent → handled by the torrent engine.
  • Any other http(s) link → fetched by the direct-link downloader.

Add several at once (separated by spaces/newlines), or drag-and-drop .torrent files onto the page. Tick add paused to choose which files to download before it starts. The Torrents and Downloads tabs list everything, with a filter box and live stats.

Streaming & files

Each torrent expands to its file list. For each file you can:

  • ▶ Play — opens the in-app player; video/audio streams and seeks even before the download finishes, with subtitle tracks from sibling .srt files.
  • ⤓ Download — pull the file over HTTP (supports resuming / range requests).
  • ⧉ Link — copy a direct link to open in VLC, Infuse, etc.
  • Checkboxes — choose exactly which files download; toggle anytime.

Users & invites

MagnetBox is built for a small, trusted group. Registration is closed by default.

  1. In Admin → Access, flip Registration open on during a window.
  2. In Admin → Invites, generate an invite code (single/multi-use, optional expiry) and share it.
  3. The invitee signs up at /register with the code, username, email and password.
  4. Close registration again when the window's over.

Admins can ban/unban, reset passwords, rotate a user's API token, and see last-seen/IP and bandwidth per user.

Admin console

The /admin console (admin accounts only) has:

  • Overview — live download speed, torrent/download counts, storage used, users, sessions, and host CPU / RAM / disk; quick bulk actions.
  • Torrents / Downloads — every item with controls and bulk pause/resume/clear.
  • Users / Invites — account and invite management.
  • Sessions — every active login, revoke any.
  • Activity — an audit log of state-changing actions.
  • Access / Settings — registration window, maintenance mode, speed limits, per-user concurrency, retention.

Retention (auto-expiry)

To keep disk usage in check, set Admin → Access → Auto-delete after (days). MagnetBox then removes torrents and downloads (and their files) older than that, on an hourly sweep, with a Run cleanup now button for manual runs. 0 = keep forever.

API & tokens

Every endpoint can be driven programmatically. On the Account page, generate an API token, then call the API with:

curl -H "Authorization: Bearer YOUR_TOKEN" https://your-host/api/torrents

The full machine-readable spec is at /api/openapi.json (OpenAPI 3.1) and a live reference page is at /docs on your running instance — import it into Postman/Insomnia or point Kodi/Stremio-style tools at it.

Security

  • argon2-hashed passwords; 2FA (TOTP) with one-time recovery codes.
  • Sessions over HttpOnly + SameSite=Strict cookies (Secure with MAGNETBOX_HTTPS=1).
  • SSRF protection on the link downloader (refuses private/loopback/cloud-metadata addresses, including across redirects).
  • Security headers (nosniff, X-Frame-Options: DENY, etc.); CSRF protection; login throttling.
  • Secrets files written owner-only (0600) on Linux.
Before going live: run behind HTTPS, keep registration closed, enable 2FA on admin accounts, firewall to 80/443 only, and run cargo audit on the server to scan dependencies for CVEs.

FAQ & troubleshooting

I lost the admin password / can't log in

Set MAGNETBOX_ADMIN_PASSWORD to a new value (≥8 chars) and restart — it resets the admin account. Watch the console for the confirmation line. Note the username is case-sensitive (default admin).

A magnet won't resolve / nothing downloads

Usually a firewall/VPN/antivirus blocking BitTorrent traffic. Allow the binary through, and try a well-seeded magnet. After 120 s the UI shows a clear message.

Registration says it's closed

That's the default. An admin opens it in Admin → Access and hands out an invite code.

Does it phone home or need your servers?

No. It runs entirely on your machine — no telemetry, no external dependency to keep working.

What can I legally use it for?

It's a neutral tool. Use it for content you have the right to download and store — your own files, Linux ISOs, Creative-Commons and licensed media. You're responsible for what you do on your own server.
